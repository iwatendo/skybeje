モバイル端末での接続について

上記の接続QRコードを読みとる事で、モバイル端末でもライブキャストを見る事ができます。
モバイル端末の標準ブラウザ（Android端末の場合はChrome / iPhoneの場合はSafari）で開いてください。
iPhoneの場合は、外部アプリでQRコードを読むのではなく、標準カメラでQRコードを読みとることをお勧めします。

アプリ内ブラウザ(WebView)から接続した場合、正常に動作しないケースがあります。
一例として、LINEのQRリーダーから起動した場合、LINEアプリ内のブラウザで起動しますが
カメラ及びマイクへのアクセス権限が与えられていない為、アプリ内のブラウザから配信はできません。
※ライブキャストの受信は問題ないようです。

ライブキャスト機能について

SkyWay を使用したP2Pでのライブキャストであり、基本的には少人数向けの配信機能です。
単方向のビデオ通話（グループ通話）のイメージの機能です。
SFUを使用する事により、ある程度の人数への配信は可能となりますが大規模配信はできません。
どの程度の人数にまでの配信に耐えれるか未検証ですが、本サービスでは10人以下での利用を想定しています。

SFUについて

SFUについての詳細は SkyWayの説明ページを参照してください。
PCや回線の性能に依存しますが、2～3以下の接続の場合はSFUは不用です。
SkyWayのSFUサーバーは東京にあり、SFUを使用する場合は一度東京にあるサーバーを経由します。
環境にもよりますが、2～3人で利用する場合はSFUを使用しない方が安定するようです。

2018年3月時点で SkyWayのSFUは「Safari」及び「iOS」に未対応です。
よって iPhone/iPad ではSFUを使った配信および受信ができません。
MacOSの場合、Google Chromeを使用すればSFUを使用できます。
iPhone/iPadの場合、iOS版のGoogle Chrome を使用してもSFUを使用できません。

ライブキャストができない組合せについて

WebRTCは V8/V9/H264 等の映像コーディックが使用されます。
配信する側と受信する側で、一致する利用可能な映像コーディクが無い場合、ライブキャストできません。
現在、SafariがH264のみ対応しています。
Androidの場合、H264に対応していない端末が存在します。

そのため、以下のような組合せではライブキャストができません。
・H264未対応のAndroid端末 と iPhoneまたはiPad
・H264未対応のAndroid端末 と MacOSでSafariを使用（※Chromeの場合は大丈夫です）
