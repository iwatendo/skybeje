モバイル端末での接続について

接続QRコードを読みとる事で、モバイル端末でもライブキャストが視聴できます。
・iPhoneの場合、標準カメラでQRコードを読取ってください。
・Android端末の場合、QRコード読みとり後、Chromeで開いてください。
※アプリ内ブラウザ(WebView)から接続した場合、正常に動作しないケースがあります。

モバイル端末でのライブキャストについて

アカウント登録や外部アプリ不用で、ブラウザのみで配信できます。

モバイル端末側で「開始」ボタンを押すとライブキャストが開始されます。
チャットクライアントから呼出した場合、自動的にチャットルームで視聴ページが開きます。
単体で起動した場合は、接続URLをコピーするためのボタンが表示されますので、視聴して欲しい人にURLを伝えてください。

一般的なビデオ通話に使用されるWebRTC(P2P)を使用した配信であり、大規模配信には利用できません。
本サービスは、10人以下での利用を想定しています。

SFUについて

本サービスはNTT Communicationsの SkyWayを使用しています。

多人数への配信を安定させる為に、SkyWayのSFUを利用できます。
詳細については SFUについてを参照してください。

1～2人への配信する場合は基本的にSFUは不用です。
3～4人以上の場合で、配信が安定しない場合はSFUを使用してください。

SkyWayのSFUサーバーは東京にあります。その為SFUを使用した場合、東京のサーバーを経由します。
環境にもよりますが、SFUを使用しない方が安定するケースもあるようです。

2018年3月時点で SkyWayのSFUは「Safari」及び「iOS」に未対応です。
そのため、iPhone/iPad ではSFUを使った配信および視聴ができません。

MacOSの場合、Google Chromeを使用すればSFUを使用できます。
但しiPhone/iPadの場合、iOS版のGoogle Chromeを使用してもSFUは使用できません。

ライブキャストができない組合せについて

WebRTCは V8/V9/H264 等の映像コーディックが使用されます。
ブラウザや端末によって利用できる映像コーディックに差異があります。
配信側と視聴側で、利用できる映像コーディクで一致するものが無い場合、視聴できません。

現在、SafariがH264のみ対応で、Androidの場合H264に対応していない端末が存在します。
そのため、以下のような組合せではライブキャストの視聴ができません。
・H264未対応のAndroid端末 と iPhoneまたはiPad
・H264未対応のAndroid端末 と MacOSでSafari（※MacOSでもChromeの場合は大丈夫です）

上記以外で配信または視聴ができない場合

トラブルシュートを確認してみてください。
